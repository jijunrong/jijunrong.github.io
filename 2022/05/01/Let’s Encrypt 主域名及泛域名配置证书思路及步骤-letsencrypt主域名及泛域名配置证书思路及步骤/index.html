Let’s Encrypt 主域名及泛域名配置证书思路及步骤

发表于2022-05-01更新于2022-05-01
地球

本文按dns思路配置泛域名证书。

条件：
（1）Cloudflare域名已配置二级域名。
（2）获取Cloudflare API 密钥：首先点击我的个人资料 -> API 令牌。
查看 Global API Key复制即可。

1.安装acme：
curl https://get.acme.sh | sh

2.添加软链接：

ln -s /root/.acme.sh/acme.sh /usr/local/bin/acme.sh

3.切换CA机构： (可换可不换，习惯使用let)

acme.sh –set-default-ca –server letsencrypt

4.配置cloudflare变量

export CF_Email=”XXXX@gmail.com“
export CF_Key=”XXXXXXXXXXXX”

5.申请证书

泛域名证书

acme.sh –issue –dns dns_cf -d “*.jijunrong.site” –keylength ec-256

主域名证书

acme.sh –issue –dns dns_cf -d jijunrong.site –keylength ec-256

若报错，Google错误，或留言。

6.复制证书出来并保存（可跳过，一般建议复制出来）
根据上步最终的路径，修改前半部分位置，后半部分可自定。

cp /root/.acme.sh/.jijunrong.site_ecc/.jijunrong.site.cer /root/server.crt

cp /root/.acme.sh/.jijunrong.site_ecc/.jijunrong.site.key /root/server.key

7.配置证书位置到所需主域名或泛域名证书的服务。
